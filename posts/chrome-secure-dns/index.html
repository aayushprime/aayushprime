Chrome's Secure DNS: What is it and how to use it

February 5, 2026

Today I found out that Google Chrome has a feature called “Secure DNS” that allows users to specify a custom DNS provider (also options to choose from popular ones like Cloudflare). Some time ago, I must have turned this on or maybe it was enabled by default.

Today I was trying to visit callbreak.games and mistakenly went to callbreak.bhoos.games. And what do I see? I was redirected to this random website planet.news. I thought there was a bug somewhere or I downloaded some kind of malware. I tried disabling all the extensions in my Chrome and I was still getting the same redirect happening.

In the end I found out that my system DNS was alright. Along the way I found out Chrome has its own internal net log viewer, DNS tools available. I tried everything, recording chrome logs, trying to look at extensions code and one particular extension Control for Instagram was particularly suspicious because if I turned it off the redirect would disappear and if I enabled it, the redirect was there again. (It was a fluke)

I queried Chrome’s DNS (using chrome://net-internals/#dns) and it was showing some random IP address range and it was pointing to the DNS. It was doing some kind of weird thing where it would request some JavaScript from the server and then try to hit an endpoint and if the endpoint responded with 200, then it would redirect me to some another domain which for me was planet.news.

The reason for all this is actually really stupid. The Chrome Secure DNS feature is actually not what I would call secure at all. It uses DNS over HTTP and when you query the DNS over HTTP and the DNS response is NXDOMAIN which means it does not actually exist. We can see this in the dig command in the system using the system DNS.

So somewhere along the return path, maybe the ISP or some other node in the network, changed the NXDOMAIN response to some kind of ad DNS. So my domain actually resolves to the ad page and that’s why I was getting that ad page.
